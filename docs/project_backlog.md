## Operating Rules

- Prefer shiftability and reliability over matching percentage until all three
  versions pass build, split-audit, and runtime-smoke gates.
- Use evidence-based split changes only: source order, symbols, strings, xrefs,
  generated reports, and version comparisons.
- Do not add filler assembly to preserve layout. If a shifted build fails, fix
  split ownership or symbol labels instead.
- Time-box work by task type:
  - Build/config failure: 60 minutes.
  - Split-port batch: 2 hours.
  - Runtime-smoke failure: 2 hours.
  - Matching attempt: 45 to 120 minutes, based on function size.
- If the time box ends without a clear next step, park the item here with exact
  commands, artifacts, hypotheses tested, and the smallest known next question.
- Do not revisit a parked item until at least two other tasks have landed or the
  weekly backlog pass reaches it.
- Absolute failsafe: after 3 failed independent passes on the same item, mark it
  `human-review-required`, stop attempting it, and leave a minimal reproduction
  plus notes for human review.

## Status Values

- `ready`: enough evidence exists for the next pass to begin.
- `in-progress`: actively being worked.
- `parked`: time box expired; wait for the revisit rule.
- `human-review-required`: 3 independent failed passes; do not retry without
  human input.
- `done`: verified by the gate listed in the evidence column.
